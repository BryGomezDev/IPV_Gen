FAQ – Generador de comandos

Guía rápida para entender y usar la herramienta

Este documento responde a las preguntas más frecuentes sobre el uso de la herramienta de generación de comandos para las distintas regulaciones (España, Portugal y Colombia).

1. ¿Qué es esta herramienta?

Es una aplicación web local que te permite construir comandos automáticamente para ejecutar distintos procesos reguladores sin tener que escribirlos a mano.

Solo eliges las opciones que necesites, pulsas Generar comando, y el panel de salida mostrará uno o varios comandos listos para copiar.

2. ¿Qué regulaciones soporta?

Actualmente la herramienta está preparada para trabajar con:

  • 🇪🇸 España
  • 🇵🇹 Portugal
  • 🇨🇴 Colombia

Cada regulación tiene su propio fichero HTML con su lógica específica y se accede a ellos desde el index.html.

3. ¿Qué tipos de generación de comandos existen?

Daily (Diario)

Genera un comando por cada día dentro del rango indicado (fecha inicio → fecha fin). Es ideal para procesos que trabajan con días completos.

Monthly (Mensual)

Genera un comando por mes, tomando como referencia siempre el día 1 (por ejemplo, 2025-01-01, 2025-02-01…). Es útil para informes o procesos mensuales.

Horario (por horas)

Permite generar comandos para rangos de horas específicos. En la regulación de España se usan rangos dentro de 0–24. La herramienta valida que el rango horario tenga sentido antes de generar el comando.

Además, en este modo se activa la selección de GameType para la regulación española, concretamente para los ficheros JUC.

4. ¿Para qué sirve el apartado “Formato” al generar comandos?

El control de Formato determina cómo se muestran los comandos en el panel de salida y está pensado específicamente para adaptarse a distintos métodos de ejecución dentro del entorno de trabajo.

🔹 1) Job Automático SCI - Multicomandos

Este formato envuelve cada comando entre comillas simples ' ' y añade una coma al final. Está pensado para usarlo cuando el comando va a formar parte de un array o una lista de comandos que será procesada por nuestro job automático de ejecución de comandos.

Ejemplo: 

'C:/ruta/app.exe --date 20250101',

🔹 2) Job Automático SCI - Único comando

Similar al formato anterior, pero sin la coma final. Este modo se usa cuando el job automático espera un único comando dentro del listado. Es decir, cuando solo se va a ejecutar 1 comando en lugar de varios.

Ejemplo:

'C:/ruta/app.exe --date 20250101'

🔹 3) Para ejecutar por CMD

Genera los comandos tal cual deben ejecutarse en una ventana de CMD / PowerShell local. Es el formato más directo y limpio, pensado para copiar y pegar en consola durante pruebas o ejecuciones manuales.

Ejemplo:

C:/ruta/app.exe --date 20250101

📌 ¿Cuál debería usar?

  • Automatización con múltiples comandos: Con comillas + coma.
  • Automatización con un único comando: Con comillas sin coma.
  • Ejecución manual en consola: Sin comillas ni coma.
5. ¿Para qué sirve cada campo del formulario?

Ruta ejecutable

Muestra la ruta del .exe que se va a utilizar en el comando. El campo es solo lectura para evitar modificaciones accidentales. La herramienta ya se encarga de generar el comando con la ruta correcta.

Fechas (inicio y fin)

  • En modo Daily, se genera un comando por cada día del rango.
  • En modo Monthly, se genera un comando por cada mes del rango.
  • En modos Horario, normalmente se utiliza un solo día combinado con un rango horario.

Selector UTC

Permite elegir la zona horaria que se aplicará en el comando (por ejemplo, +01:00 o +02:00). La herramienta puede ajustar automáticamente este valor según la fecha o dejar que lo selecciones manualmente.

Selector de GameType

Se muestra cuando la periodicidad es Horario y JUC. Permite filtrar por uno o varios tipos de juego.

Si no seleccionas ningún GameType, el comportamiento por defecto suele ser “todos los tipos” (según la lógica de cada regulación).

6. ¿Para qué sirve cada botón?

Generar comando

Construye el comando (o comandos) según las opciones seleccionadas: fechas, periodicidad, rangos horarios, GameType, etc.

Si el rango abarca varios días o meses, se generará un comando por cada día/mes correspondiente. Por defecto, los nuevos comandos se añaden al final del panel de salida sin borrar los anteriores.

Copiar

Copia el contenido completo del panel de salida al portapapeles.

Limpiar

Borra por completo el área de salida de comandos. No modifica las opciones del formulario, solo limpia el resultado generado.

Volver al índice

Te lleva de vuelta al menú principal, donde puedes escoger una nueva herramienta.

7. ¿Cómo se construyen los comandos internamente?

Aunque el formato exacto puede variar entre regulaciones, la estructura general de un comando suele seguir este patrón: 

<ruta_del_exe> --f <frecuencia> --date <YYYYMMDD> [opciones adicionales]

En las opciones adicionales se pueden incluir:

  • Rangos horarios (en modos Horario/JUC).
  • GameType (uno o varios tipos de juego).
  • Zona horaria (UTC) u otros parámetros específicos de la regulación.

Si seleccionas múltiples días, meses o GameTypes, la lógica de la herramienta genera automáticamente tantos comandos como sea necesario.

8. ¿Qué sucede si hay un error en los datos?

Antes de generar los comandos, la herramienta ejecuta una función de validación. Si detecta algún problema, se lanza un mensaje de error claro, por ejemplo:

  • «Indica fecha inicio y fin.»
  • «Fin anterior a inicio.»
  • «Rango horario inválido.»
  • «Selecciona periodicidad.»

Corrige el dato que se indica en el mensaje y vuelve a pulsar Generar comando.

9. ¿Se pueden acumular varios comandos en pantalla?

Sí. Cada vez que pulsas Generar comando, los nuevos comandos se añaden al final del panel de salida, sin eliminar los anteriores.

Esto permite preparar varios bloques de ejecuciones distintas y copiarlos todos de una sola vez. Para reiniciar la salida, utiliza el botón Limpiar.

10. ¿Necesito servidor web para usar la herramienta?

No. La herramienta está pensada para ejecutarse en local. Solo necesitas:

  • index.html y los HTML de cada regulación.
  • El fichero de estilos CSS común.
  • Los ficheros JavaScript con la lógica de cada regulación.

Basta con abrir el index.html con doble clic en tu navegador y desde ahí navegar a cada regulación o a esta página de FAQ.

11. ¿Se puede ampliar o modificar la herramienta?

Sí. La estructura del proyecto está pensada para ser extensible:

  • Añadir nuevas regulaciones con su propio HTML y JS.
  • Agregar nuevos tipos de generación de comandos.
  • Modificar validaciones, rutas de ejecutables, parámetros, etc.

Siempre que mantengas el mismo estilo de CSS y la misma estructura básica, la experiencia de uso será consistente con el resto de páginas.
